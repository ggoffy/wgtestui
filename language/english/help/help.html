Help: wgTestUI

Description

The module wgTestUI module can be used to test the modules of your XOOPS

Install/uninstall

No special measures necessary, follow the standard installation process and extract the wgtestui folder into the ../modules directory. Install the module through Admin -> System Module -> Modules.

Detailed instructions on installing modules are available in the XOOPS Operations Manual

Features

What can wgTestUI do?
Test XOOPS sites for:
  • Fatal errors (site blocked)
  • Errors (notices, warnings)
  • Deprecated functions
What can wgTestUI (currently) not do?
  • Execute actions, e.g. save data, delete data,...
  • Simulate user actions, e.g. type in data in a form

The wgTestUI module continues to expand, to get to the conditions to create modules, more and more sophisticated.
For this reason, I invite all developers to report and send in svn any changes or additions to this module, so that we can jointly contribute to the development

Tutorial

Create the list of links for testing
  • You can type in one by one with 'Add New Test'
  • For collection of urls of user site, please use the block 'Block add new tests', activated it for all pages, then it goes very quickly
    Simply go from site to site and click always only on button 'Add current url to test database'
  • Use 'Datatools'
Execute tests

Before starting the tests you should:

  • Enable debug mode inline, in order to get information about errors, notices and warnings
  • If you want to analyse a hugh number of urls then it is maybe necessary to increase the value for "maximum execution time" in your php settings

If you execute the analysis, only tests with "Result code = 0" will be executed. If you want to re-analyse all uls then you have to reset test results before.

Execute tests for area ADMIN and/or USER:
If you what to test another new theme then it makes no sense to test also the admin area, therefore you can start only 'Execute Tests USER'

If you have loaded module tests and this module is not installed then this test will be skipped.

Result of tests

The result shows you, how many findings were made (fatal errors, warnings,...). In 'Info text' you can see the information about the findings

If you want you can filter the list by module, errors and so on

Statistics

When clicking on 'Statistics' you get an overview about all loaded module test and the sums of errors, information and so on.

Datatools

With datatools you can do:

  • Import data about lists of tests from a json file in folder 'datatools'
  • You can also export data from lists of tests into a json file in folder 'datatools'
  • You can collect the links in 'Import as list' and add them at once.
    Use a new line for each new url.
  • If your module follows the structure of Modulebuilder you can also use the generator in datatools
    The generator reads the admin/menu.php of the module and creates a list of possible links.
    If you like the list then copy it into the form 'Import as list'.
